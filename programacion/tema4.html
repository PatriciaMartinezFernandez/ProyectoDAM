1. Bucle while


  1. Iteraciones Indeterminadas: El bucle permite repetir un conjunto de instrucciones varias veces, pero no se especifica previamente cuántas veces o en qué momento se detendrá.

  2. Desconocimiento Previo: A diferencia de algunos bucles que requieren conocer el número exacto de repeticiones antes de su ejecución, este tipo de bucle no exige ese conocimiento previo.

  3. Expresión Booleana de Control: A diferencia de algunos bucles que requieren conocer el número exacto de repeticiones antes de su ejecución, este tipo de bucle no exige ese conocimiento previo.

  4. Modificación de la Expresión Booleana: Se requiere que al menos una sentencia dentro del bucle modifique algo para que, en algún momento, la expresión booleana se evalúe como falsa, asegurando la eventual terminación del bucle.

  5. Declaración e Inicialización de Variables: Las variables de la expresión de control deben ser declaradas e inicializadas antes del bucle, garantizando que el bucle tenga acceso a valores concretos desde el principio.

  6. Sintaxis del Bucle while:

                while (condición) {
                //acciones a ejecutar
             }

2. Bucle do-while


  1. Iteraciones Indeterminadas: Permite repetir un bloque de código un número indeterminado de veces sin saber previamente la cantidad de repeticiones.

  2. Variante con respecto al Bucle while: Se destaca que, a diferencia del bucle while convencional, el bucle do-while garantiza que al menos se ejecuta una iteración, ya que la condición se evalúa después de ejecutar el bloque de código.

  3. Expresión Booleana de Control: Utiliza una expresión booleana que, mientras sea verdadera, mantiene la ejecución en el bucle.

  4. Modificación de la Expresión Booleana: Al igual que en el bucle while, se requiere que al menos una sentencia dentro del cuerpo del bucle modifique algo para que, en algún momento, la expresión booleana se evalúe como falsa, asegurando la eventual terminación del bucle.

  5. Declaración e Inicialización de Variables: Las variables de la expresión de control deben ser declaradas e inicializadas antes del bucle, asegurando que el bucle tenga acceso a valores concretos desde el principio.

  6. Sintaxis del Bucle do-while:

                do {
                //acciones a ejecutar
            } while (condición);

3. Bucle for


  1. Iteraciones Determinadas: Permite repetir un bloque de código un número determinado de veces, conocido de antemano.

  2. Declaración/Inicialización:

    • Se ejecuta una sola vez al inicio del bucle.

    • Se utiliza para declarar y/o inicializar una o más variables que gestionarán el bucle.

    • Si se declara la variable aquí, su alcance se limita al bucle.


  3. Condición:

    • Es una expresión booleana que determina si se debe continuar en el bucle o salir de él.

    • La condición de continuación se evalúa antes de cada iteración; si es verdadera, se realiza una iteración más.


  4. Incremento/Decremento:

    • Se ejecuta después de ejecutar las acciones del cuerpo del bucle, en cada iteración.

    • Incrementa o decrementa las variables de control declaradas o inicializadas en el propio bucle.

    • Puede realizar incrementos o decrementos en más de una unidad.


  5. Sintaxis del Bucle for:

                    for (declaración/inicialización; condición; incremento/decremento) {
                    //acciones a ejecutar
                }

4. Recomendaciones


Se recomienda el uso del bucle for cuando se conoce el número exacto de repeticiones, como en la impresión de un mensaje 100 veces. Para situaciones donde el número de repeticiones es desconocido, se prefiere el bucle while, como al leer números hasta que la entrada sea 0. El bucle do-while es útil como reemplazo del bucle while cuando es necesario ejecutar el cuerpo antes de evaluar la condición de continuación. La elección entre estos bucles depende de la claridad y la lógica específica del código.
